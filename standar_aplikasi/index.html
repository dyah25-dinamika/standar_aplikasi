Standar Tata Kelola Aplikasi Mahasiswa

IT Server Deployment Guidelines v1.0

📋 Pendahuluan

Dokumen ini menetapkan standar kualitas aplikasi yang wajib dipenuhi mahasiswa sebelum Deploy ke server IT. Standar ini bertujuan memastikan:

  • Keamanan sistem server tetap terjaga
  • Aplikasi memiliki kualitas kode yang baik dan maintainable
  • Terhindar dari kerentanan keamanan (SQL Injection, XSS, CSRF, dll)
  • Dokumentasi lengkap untuk proses debugging dan maintenance
  • Konsistensi struktur project antar mahasiswa

🛠️ Stack Teknologi yang Didukung

Laravel Python PHP Native JavaScript

✅ Checklist Deploy Wajib

  • Aplikasi sudah diuji di environment lokal (localhost)
  • Semua dependency sudah ter-install dengan benar
  • File konfigurasi environment (.env) sudah dikonfigurasi
  • Database migration/structure sudah ready
  • Tidak ada credential/test data hardcoded
  • Error logging sudah aktif
  • Cache sudah dikonfigurasi dengan benar

    • 🔒 Standar Keamanan (WAJIB)

    Kategori Persyaratan Status
    SQL Injection Semua input wajib menggunakan parameterized queries / ORM WAJIB
    XSS Output encoding pada semua user input, gunakan sanitization WAJIB
    CSRF Implementasi CSRF token untuk semua form WAJIB
    Authentication Password hash dengan bcrypt/argon2, salt unik WAJIB
    Session HttpOnly, Secure, SameSite cookies WAJIB
    File Upload Validasi tipe file, simpan di luar webroot WAJIB
    API Security Rate limiting, JWT/explicit auth REKOMENDASI
    SSL/TLS Konfigurasi HTTPS dengan SSL valid WAJIB

    📝 Standar Kode per Stack

    Laravel

    • Gunakan struktur folder Laravel standar (app/Http/Controllers, Models, dll)
    • Gunakan Eloquent ORM untuk database operations
    • Validasi request dengan Form Requests
    • Gunakan Laravel Mix/Vite untuk asset compilation
    • Konfigurasi environment menggunakan .env (bukan hardcode)
    • Gunakan Laravel logging untuk error tracking

    Python

    • Gunakan virtual environment (venv) untuk isolate dependencies
    • Gunakan requirements.txt atau poetry/conda untuk dependency management
    • Struktur project: app/, config/, models/, routes/
    • Gunakan ORM (SQLAlchemy/Django ORM) untuk database
    • Pisahkan config dari kode (environment variables)

    PHP Native

    • Gunakan struktur MVC (Models, Views, Controllers)
    • Gunakan Prepared Statements untuk SQL queries
    • Pisahkan config (database.php, config.php)
    • Gunakan autoloader untuk class loading
    • Hindari penggunaan eval() dan inline PHP
    • Pisahkan business logic dari presentation layer

    JavaScript

    • Gunakan Node.js LTS version
    • Package.json dengan script yang jelas
    • Gunakan ESLint dan Prettier untuk code formatting
    • Pisahkan config dari kode (dotenv)
    • Backend: Express.js dengan struktur route/controller
    • Frontend: Build tool (Vite/Webpack), modular components

    📚 Standar Dokumentasi

    Dokumen Deskripsi Wajib
    README.md Cara install, setup, dan cara jalankan aplikasi WAJIB
    .env.example Template variabel environment (tanpa nilai sensitif) WAJIB
    Database Schema ERD atau migration file WAJIB
    API Documentation Endpoint list, method, parameter, response REKOMENDASI
    Changelog Riwayat perubahan versi REKOMENDASI

    🔍 Proses Review Deploy

    1. Submit Aplikasi: Mahasiswa submit aplikasi ke repository yang ditentukan
    2. Review Awal: Admin IT cek kelengkapan file dan struktur
    3. Security Scan: Scanning kerentanan menggunakan tools otomatis
    4. Code Review: Review kode oleh team IT
    5. Deploy Test: Deploy ke staging environment untuk pengujian
    6. Approval: Jika semua standar terpenuhi, approval deploy ke production

    ⚠️ Larangan

    • DILARANG Menyimpan credential di repository (.env的真实 value)
    • DILARANG Menggunakan password default/admin untuk aplikasi
    • DILARANG Menyimpan file sensitif di public directory
    • DILARANG Men-deploy aplikasi tanpa testing
    • DILARANG Mengaktifkan debug mode di production
    📌 Catatan Penting:
    Aplikasi yang tidak memenuhi standar ini TIDAK AKAN di-deploy ke server production. Mahasiswa bertanggung jawab penuh untuk memastikan aplikasi memenuhi semua persyaratan sebelum mengajukan deploy.
